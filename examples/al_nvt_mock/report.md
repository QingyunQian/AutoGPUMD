# AutoGPUMD Workflow Report

**Data mode: MOCK.**

## Overview

AutoGPUMD is an unofficial learning/demo workflow layer around GPUMD/NEP.
It automates lightweight setup, execution, analysis, plotting, and Markdown reporting.

## Data provenance

This report uses deterministic synthetic mock data generated by AutoGPUMD. These data are useful for workflow testing only and are not physical GPUMD/NEP simulation results.

## System

- Structure input: `structure.xyz` when present.
- Run template: `run.in` when present.
- Potential: user-provided `nep.txt` is required for real GPUMD runs.

## Simulation settings

The MVP records settings in `run.in` comments and keeps GPUMD syntax validation outside scope.

## Limitations

- This is not a replacement for GPUMD, gpyumd, GPUMDkit, or official GPUMD tooling.
- Mock data are synthetic and intended only for workflow validation.
- The parser supports AutoGPUMD mock CSV and simple XYZ/extended XYZ trajectories.
- Real NEP potentials are never fabricated by this project.
